Support

Public support routes for Cadence.

Cadence currently uses GitHub as the primary channel for bug reports, release troubleshooting, and product feedback.

Open the right thread

Report a bug Use the issue form and include the reproduction path. Check release assets Confirm the tag, installer, and checksum before reporting. Browse the repository Inspect source, docs, workflows, and current project state.

Include this in a bug report

  • The app version or release tag you tested.
  • Your operating system and architecture.
  • What you expected to happen.
  • What actually happened.
  • Whether the issue reproduces after restarting the app.

Before posting publicly

Do not include secrets, passwords, tokens, private keys, or personal data in public issues. If the problem is strictly about a release asset, reference the tag and the exact filename that failed.
