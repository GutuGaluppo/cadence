Privacy

Local-first by product design.

Cadence is built around local persistence for core functionality and does not require an account for the current product scope.

What the app stores

  • Task data in a local SQLite database on the device.
  • Settings and preferences in local storage used by the desktop shell.

What the app does not require

  • No account or sign-in.
  • No cloud sync in the current release path.
  • No bundled analytics or ad tracking scripts in this repository.

Public website and external links

The landing page is static and links to GitHub for downloads, release notes, and support. If you use those external services, they may process standard request metadata such as IP address, user agent, and access logs under their own policies.

Reporting issues safely

If you open a public issue, do not include secrets, passwords, private keys, or personal data that should not be publicly visible.
