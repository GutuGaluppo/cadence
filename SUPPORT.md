# Support

Cadence currently uses GitHub as the primary public support channel.

## Where to report issues

- bug reports: `https://github.com/GutuGaluppo/cadence/issues/new/choose`
- release history and installer assets: `https://github.com/GutuGaluppo/cadence/releases`
- repository source: `https://github.com/GutuGaluppo/cadence`

## Before opening an issue

Check the following first:

- confirm you are testing the latest release or the current branch head
- verify whether the problem is already listed in open issues or recent release notes
- include your operating system, architecture, app version, and a deterministic reproduction path

## What to attach

High-signal reports usually include:

- the exact app version or Git commit
- the operating system and architecture
- what you expected to happen
- what actually happened
- whether the issue reproduces after restarting the app

## Sensitive information

Do not post secrets, tokens, passwords, or personal data in public issues.

## Release support scope

The public download channel is GitHub Releases. If a release asset is missing, unsigned, corrupted, or has a checksum mismatch, open an issue and reference the tag name and asset filename.
